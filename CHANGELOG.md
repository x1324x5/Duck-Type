# 更新日志 · Changelog

本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [Unreleased]

### Added 新增
- **报告 & 分享卡片**：今日小结 / 周报 / 月报 / 年度报告，后端用 Pillow 生成精美 PNG
  卡片（含字数、高峰时段、最常用词/字、主力应用占比、最高产日、最长连续输入、年度关键词等），
  一键保存，适合发朋友圈 / X / GitHub README。
- **检查更新**：设置页新增「检查更新」，对比 GitHub Releases；打包版支持「一键更新并重启」
  （下载新 exe → 退出时自动替换并重启）。
- **成语单列**：趣味页把「成语」与「长词」分开统计。
- **词性/主题等信息说明**：面板标题旁的 ⓘ 图标，鼠标悬停解释 `uj/ul/ug…` 等词性标签含义。
- **刷新按钮**：无需刷新浏览器即可重新拉取当前页数据。
- **更鲜活的横幅**：捕获正常时轮播"鸭子小贴士"（今日字数 / 目标进度 / 连续天数 / 高峰时段…），
  仅在异常时显示排查信息。
- **UI 美化**：渐变背景、毛玻璃顶栏，顶部与网页图标改用自定义鸭子图（`/favicon.png`）。

### Fixed 修复
- **退出时让微信/QQ 在 MSCTF.dll 崩溃**：注入的 DLL 现在自我钉住（pin），不再在进程运行中
  被卸载，避免 TSF 回调进入已卸载代码导致宿主程序崩溃。
- **粘贴保护**：单次插入超过约 30 字视为粘贴/程序插入，不计入输入统计，避免污染字数与报告。

### Added（capture）
- **TSF 捕获引擎**：原生钩子改用 C++/COM，新增 `ITfThreadMgrEventSink` /
  `ITfTextEditSink` 监听 TSF 文本提交，从而能记录搜狗/微软拼音在**现代程序**
  （微信、VS Code、Win11 记事本、浏览器…）里的上屏中文；经典 `WM_CHAR` 通路保留并互斥，
  不重复计数。DLL 静态链接，注入时自包含。
- **健康横幅 + `/api/status`**：仪表盘顶部实时显示钩子状态（DLL 是否找到 / 是否安装 /
  收到字符消息数 / 已捕获汉字数），并据此给出排查指引。
- **网页图标用自定义鸭子**：新增 `/favicon.png`，与托盘图标一致。
- 🦆 全新 **DuckType（码字鸭）** 品牌与卡通鸭图标（由 `branding.py` 代码绘制，`tools/make_icon.py` 生成 `.ico`）。
- **输入序列浏览与导出**：仪表盘「序列」标签按时间线查看上屏汉字，支持导出 TXT / CSV / JSON。
- **数据管理与隐私**：一键清空、按日期区间删除、`retention_days` 自动保留期限（后台定时清理）。
- **网页设置页**：在仪表盘里直接编辑黑名单、目标、阈值、端口、自启等，免改 `config.json`。
- **自定义时间范围**：除「今天 / 7 天 / 30 天 / 全部」外，可选任意起止日期统计（周报/月报即范围特例）。
- **趋势环比**：关键指标与上一等长周期对比，卡片显示升降幅度。
- **目标 / 连续天数 / 成就**：每日目标进度环、连续打卡 streak、成就徽章墙。
- **趣味榜单**：最爱词、成语/长词、生僻字、只打过一次的字。
- **CLI**：`--clear` 清空数据；`--export` 现在一并导出输入序列 `sequence.txt`。
- 测试套件（`tests/`）+ GitHub Actions 跨平台测试任务。

### Changed 变更
- 数据目录迁移至 `%APPDATA%\DuckType`（数据库 `ducktype.db`、日志 `ducktype.log`）。
- 原生钩子 DLL 更名为 `ducktype_hook.dll`，窗口类/消息名随之更新。

### Fixed 修复
- **单实例保护**：用命名互斥量防止开机自启 + 手动启动产生两个实例互抢钩子与端口。
- **端口占用**：仪表盘端口被占用时自动顺延到邻近可用端口，而不是启动崩溃。
- 移除键盘钩子中未接线的死代码 `on_any`。
