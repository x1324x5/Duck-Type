# 更新日志 · Changelog

本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [Unreleased]

## [0.1.7]

### Added 新增
- **语录相关成就**：新增与看板滚动语录相关的成就——累计看过语录的次数（200 / 1,000 次），读过的不同
  语录条数（50 / 200 / 500 条），以及一个隐藏的「一片留白」彩蛋：随机滚动到那条空白语录即可解锁。
  语录浏览以新表 `quote_views` 记录在本地数据库中。
- **小鸭子点缀随机化**：`assets/duck_positioned_assets_v2/` 下新增 25 张定位后的小鸭子插画；滚动横幅与页脚的鸭子改为随机出现，
  换一句或刷新页面时会变换，细节更灵动。

### Changed 调整
- **词性分布可隐藏「其他」**：下钻查看某一词性时新增「显示『其他』」开关——「其他」类常占很大比例、
  拉低观感，可一键隐藏，选择会被记住。
- **信息文案优化**：重写了生僻字、主题关键词、成语等板块的说明与若干栏目标题，去掉了面向开发者的术语
  （jieba / TF-IDF /「需要安装」等），更适合普通用户阅读。

### Fixed 修复
- **数据库文件丢失不再静默失败**：若运行期间数据库文件被误删，过去会因数据表缺失导致看板空白且毫无提示；
  现在会自动重建数据库，并在看板顶部明确提示数据库文件丢失、历史数据可能已无法找回。

## [0.1.5]

### Added 新增
- **主题偏好持久化**：仪表盘新增「跟随系统 / 亮色 / 暗色」三种主题模式，保存到
  `%APPDATA%\DuckType\config.json`，新开浏览器页面会沿用上次选择。
- **语录库精选重做**：看板滚动文字改为人工精选的近现代中外名句库，约 510 条、覆盖约 197 位作者/作品，
  主题围绕爱情、孤独、思考与命运感（英文约占 15%），每条均带作者/作品出处。相比旧版按整本作品逐句
  抓取（作者集中、混入叙述性废句），新库以「确为名句、出处可靠」为准，剔除常见的网络误传句（如伪托
  《百年孤独》《活着》的鸡汤句）。`tools/build_quote_bank.py` 由抓取脚本改为校验脚本（查重、出处、
  中英占比与作者覆盖）。
- **鸭子素材**：仪表盘可加载仓库根目录 `assets/` 下的额外鸭子图片，底部本地数据提示与语录横幅会使用
  新素材丰富视觉。

### Changed 调整
- **语录刷新节奏**：滚动横幅默认 60 秒自动换一句，并可在设置里修改；用户手动点击换一句后会重置计时，
  等完整一个周期后才自动刷新。
- **生僻字判定放宽**：在 3,500 常用字表外叠加现代常见补充字，避免「哦」「蔡」「噻」等常见语气词、
  姓氏或外来词用字误入生僻榜。
- **日期筛选与刷新按钮 UI**：自定义日期选择改为更清晰的起止日期控件，刷新按钮换成线性图标。
- **语录字体美化**：滚动横幅的语录文字改用衬线字体（英文 Georgia，中文宋体/思源宋体一类），
  并加大字号、放宽行距，呈现更接近书卷的观感，不再是死板的界面无衬线体。

### Fixed 修复
- **报告卡片重叠**：限制报告卡统计行数量、压缩关键词行并截断过长文本，避免关键词与底部 DuckType 标识
  重叠。

## [0.1.4]

### Added 新增
- **按小时输入分布图**：看板新增一张独立的「按小时」柱状图，自带时间范围切换（今天 / 近 24 小时 /

- **点击图表查详情**：点击高频字 / 高频词条形图里的任意一项，直接跳到「搜索」页看它的完整统计。
- **搜索结果更丰富**：除了总次数，新增高频字排名、占全部输入的比例、出现天数、活跃日均次数、高峰时段，
  一张「各时段分布」小图，以及若干条带高亮的「出现上下文」例句。
- **图表一键存图**：每张看板图表右上角新增 ⬇ 按钮，可把当前图表保存为 PNG。
- **GitHub 发布页入口**：顶栏右上角新增带 GitHub 图标的「发布页」按钮，一键前往
  <https://github.com/x1324x5/Duck-Type> 查看更新与源代码。
- **应用维度下钻**：点击「各应用输入量」里的某个应用，即可查看该应用内的高频词与高频字两张图，
  词 / 字均可继续点击跳到搜索详情；点「返回总览」回到应用列表。
- **看板滚动文字（ticker）**：顶部横幅改为会轮播的文字条，内容 = 程序自动生成的数据亮点（今日字数、
  连续天数、高峰时段、距下一个成就、"上次打出某个词是多久以前、之间又码了多少字"等）+ 你自己的句子；

### Changed 调整
- **词性分类更易懂、更准确**：① 词性分布图从 jieba 的约 40 个细分标签（名语素、副形词、时态助词(着)…）
  归并为 9 个**大类**（名词 / 动词 / 形容词·状态 / 副词 / 代词 / 数量词 / 时间·方位 / 虚词 / 其他），
  点进某大类再看其下的具体词语；② 修正 jieba 自带词典对部分常用词的明显误标（如「位置 / 东西」被标成
  动词 / 地名、「可以」被标成连词），用一张小型高置信度更正表纠正，更改后历史数据会自动重新分词生效
  （`seg_version`）。词性体系沿用 jieba 的 ICTCLAS（北大）标注集，本身为机器自动标注、并非 100% 准确。
- **可点击元素的悬停反馈**：标签页、按钮、主题词、可点击的条形 / 扇区等在悬停时会有边框高亮、轻微上浮
  或手型光标，"可以点" 的地方一眼就能看出来。
- **放宽「生僻字」判定**：旧版只把 CJK 扩展区 / 兼容区的字算作生僻字，几乎永远为空。现改为以
  《现代汉语常用字表》3500 常用字为基准——凡不在该字表内的汉字即视为生僻字（按标准字表判定，
  不依赖你输入了多少次），既更容易上榜，又不会把日常常用字算进去。

### Changed / Fixed 优化与修复
- **浅色主题可读性**：修复浅色模式下横幅（提示/趣味）文字沿用深色配色导致几乎看不清的问题（加粗文本
  曾是白色、警告文字接近纯白）；热力图色调、字体颜色/层级随主题正确切换，并加入字体抗锯齿，整体在
  白天模式下更清晰。
- **修复 QQ 等程序把占位文字误记为输入**：群聊被全员禁言时输入框直接显示的「全员禁言中」等占位/提示
  文字（并非用户键入）会被混进序列。根因是这类文字由控件自行写入、并未经过输入法组合（composition）。
  现在通过 TSF 组合生命周期回调（`ITfContextOwnerCompositionSink` 的 OnStart/OnEndComposition）判断
  该文档是否真的发生过输入法组合，只有发生过才采信其上屏文本；该回调在微信 / 资源管理器 /
  Electron（如 Claude）等**不暴露 composing 区段**的程序里同样会触发，因此既挡掉占位文字、又不会漏记
  这些程序的真实输入；若某程序无法挂上该回调，则自动放行（fail-open），绝不静默丢字。
- **修复更新后重复计数无需重启电脑**：旧机制每次更新都要靠改协议版本号让上一版被 pin 住的钩子失效，
  而同版本的多份钩子仍会叠加计数（开发期重复构建即出现 2×/3×）。现在每个宿主进程内的多份钩子会按
  「宿主代次」选出**唯一上报者**（宿主窗口每次启动都会换，作为代次令牌，用进程内命名共享段协调），
  宿主重启后自动重新推选——更新 / 重复注入都不再重复计数，也不必重启电脑。（同时把协议版本升到 V6
  以淘汰早于该机制的历史钩子。）
- **修复输入速度被算成天文数字**：输入法常把整词的多个字几乎同一瞬间上屏，旧算法用「相邻字时间差之和」
  作为有效时长，遇到这种成簇上屏时分母趋近 0，平均/峰值速度会暴涨到上万甚至十几万 cpm。现改为按
  「会话时间跨度」计速，并对每个字设下极小的最短耗时下限（成簇上屏时把表观速度封顶在约 500 cpm 的
  人类可达上限），正常节奏的输入完全不受影响。
- **饼图 / 条形图交互反馈**：词性饼图悬停时对应扇区会弹出并描上白色高亮环、鼠标变为手型；可点击的
  高频字 / 高频词条形悬停时整条以品牌色高亮，选择 / 点击目标更醒目。

## [0.1.3]

### Fixed 修复
- **修复拼音长词分段选择导致序列重复（`ABABCD`）**：根因是把「候选栏/预览里尚未上屏的组合文本」
  也计入了统计。整句/联想输入时先选定 `AB` 这个词，`AB` 其实仍停留在候选区、并未落到文档，
  IME 把不断增长的组合（`AB`→`ABCD`）作为编辑回报，于是被算成 `AB`+`ABCD`。现在 TSF 捕获会
  借助 `GUID_PROP_COMPOSING` 跳过仍处于组合状态的文本，**只在最终上屏（组合结束、不再 composing）
  时计入一次**，从根上消除重复，也不再需要任何「事后去重」的启发式。捕获时只枚举 composing 属性
  变化（不带 `TF_GTP_INCL_TEXT`），避免「上屏时重插文本」的输入法把同一个词记两遍。
- **修复平均/峰值速度统计**：输入法整词上屏会让一个词的每个字共用同一时间戳（我们只拿得到「上屏
  时刻」，没有逐字时刻）。旧版按字间隔算瞬时速率会被 0 间隔除爆，峰值一度高达数百万字/分。现在
  ① 峰值速度＝「任意 60 秒窗口内的最高字数」（你最高产的一分钟），有界且真实；② 平均速度＝总字数
  ÷活跃分钟，活跃分钟为会话内相邻上屏间隔之和，去掉了旧版「每个会话至少按 60 秒计」导致平均值
  系统性偏低的问题。
- **修复退格/删除率**：修改率只统计「删掉刚敲下不久的汉字」——已经停留一段时间的汉字视为“已落定”，
  之后长按退格删除其它内容（如未被捕获的英文）不再被计入删除率。
- **修复更新后重复计数（2 倍/3 倍）**：旧版钩子 DLL 会被 pin 在每个宿主进程里直到该进程退出，
  升级后若窗口/消息协议名不变，这些残留 DLL 会继续把上屏文本回传给新实例，导致同一句话被记 2~3 次。
  现在把宿主窗口类与消息名升级到 `V5`，旧 DLL 找不到新窗口即自动失效（今后改动捕获逻辑都应同步 bump）。
- **修复一键更新**：① 更新脚本原来用 `DETACHED_PROCESS` 启动、丢了控制台，`start` 拉不起新版本——
  改成隐藏控制台（`CREATE_NO_WINDOW`），用 `ping` 代替 `timeout`，并对替换动作重试（等 PyInstaller
  引导进程释放文件锁），新版本可靠重启；② 下载后校验大小（<1MB 视为重定向/错误页，直接取消，
  不会用坏文件覆盖）；③ 界面会显示新版本的**下载路径、大小与目标路径**，退出延时放宽到 3.5 秒。

### Fixed 修复
- **修复现代程序重复计数**：打包版不再从随机 `_MEI...` 临时目录注入钩子 DLL，而是使用
  `%APPDATA%\DuckType\native\ducktype_hook_<hash>.dll` 的稳定路径；同时升级 host 窗口/消息协议，
  避免旧 pinned DLL 副本继续把浏览器、微信、QQ 的 TSF 提交回传到新实例。
- **减少退出时 `_MEI...` 临时目录删除失败**：退出时显式关闭 hook/dashboard 线程，降低 one-file

- **粘贴保护**：单次插入超过约 30 字视为粘贴/程序插入，不计入输入统计，避免污染字数与报告。
