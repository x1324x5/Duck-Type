# 更新日志 · Changelog

本项目遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [0.2.9] - 2026-06-22

### Added 新增
- **里程碑通知**：达成成就、达到「今日 / 本周 / 本月」目标时，在 Windows 系统托盘弹出一条提醒（后台轮询，关着仪表盘也能收到）。设置页新增「里程碑通知」开关，可随时关闭、完全静默；已宣告过的成就 / 目标不会重复提醒（状态记在数据目录的 `notify_state.json`，首次运行只静默登记不刷屏）。
- **回顾 · 单日视图**：侧栏新增「回顾」页，挑任意一天集中复盘——当天上屏字数（含与活跃日均的对比、历史高产排名）、不同汉字 / 活跃时长 / 平均与峰值速度 / 会话 / 修改率 / 高峰时段一组数字、按小时的输入节奏柱状图、当天高频词与高频字、各应用占比，以及当天的片段节选。用 ← → 或日期框切换日期，独立于顶部时间范围。
- **记录珍藏页**：侧栏新增「记录」页，把你与码字鸭一路写下的高光时刻做成一面珍藏墙：最高产的一天、最长连续打卡、最快 / 平均速度、最长连续输入、最常写的字与词、最活跃时段、主战场应用、生僻字收藏、累计字数 / 按键、陪伴天数等，每张卡片配图标与色条。
- **每周 / 每月目标**：设置页可分别设定每周、每月字数目标（填 0 则按每日目标自动估算：周 = 日 ×7，月 = 日 × 当月天数）。看板「目标与连续打卡」面板由单个今日环升级为**日 / 周 / 月三个进度环**，各自带方向感渐变与「字数 / 目标」标注，超过 100% 同样换色重填。
- **Markdown 报告导出**：报告页卡片区新增「⬇ 导出 Markdown」，把当前周期的小结、概览表、行为洞察、高频词与高频字导出成一个 `.md` 文件，方便粘进公众号 / Notion / 周报笔记。
- **新手引导卡**：当本机几乎还没有真实数据时，看板顶部显示一张友好的引导卡（说明只数上屏汉字、如何开随身鸭、加载演示数据预览、在哪里设目标 / 自启 / 通知），可一键关闭、不再显示。

### Changed 调整
- **开机自启·状态自检**：核查并确认开机自启（写入 HKCU Run 键）实现正确——此前「重启后没自启」是因为该开关默认关闭、需在设置里手动开启。设置页现在会显示**注册表里的真实状态**（✓ 已注册 / ⚠ 已开启但未检测到，将在下次启动自动修复），与保存的偏好分开，便于核对。
- **介绍页**：新增「为什么不是数按键」对比小图（同一句话：普通统计 ≈ 12 次按键 vs DuckType 只数 3 个汉字）、常见问题 FAQ 折叠区（监控 / 隐私 / 输入法支持 / 杀软提示 / 换机迁移 / 卸载与收费）、功能速览信息标；路线图措辞淡化商业化、聚焦把本地复盘做得更好；修正隐私清单里 `.duckpack` 的代码标记。

## [0.2.8] - 2026-06-21

### Added 新增
- **看板可定制（折叠 / 拖动排序 / 隐藏）**：看板顶部新增一条工具栏。每个图表面板的标题右侧多了三个小按钮——**折叠**（只留标题、收起内容）、**隐藏**（从看板移除）、**拖动手柄**（按住即可上下拖动、重新排序面板）。点工具栏的「▦ 面板」可勾选要显示哪些面板，「↺ 重置布局」一键恢复默认顺序与显示。布局保存在本机浏览器存储里，下次打开沿用，**不影响任何统计数据**。
- **随身鸭·时间显示**：悬浮随身鸭顶部新增一个实时时钟（时:分 + 月日星期），随窗口缩小依次精简，不挤占速度表与字数。
- **仪表盘使用历史**：看板新增「仪表盘使用历史」面板，记录你每次打开**看板**或**随身鸭**的足迹：累计打开次数、看板 / 随身鸭各自次数、使用天数、最近一次时间、近 30 天每日打开柱状图（看板 / 随身鸭分色堆叠）、按小时的活跃时段分布，以及最近的打开记录。这是纯 UI 使用记录，**与打字数据无关**，删除打字数据不会清掉它（导出 / 导入备份会一并迁移）。
- **随身鸭全局热键**：设置页新增「随身鸭热键」，可分别为**打开**和**关闭**随身鸭设置全局快捷键（点输入框后直接按下组合键录制，需含 Ctrl / Alt / Shift / Win）。保存后会立即注册；若某组合**已被其它程序占用**会明确提示冲突、不会静默失效；打开与关闭设成同一组合也会被拦下。仅 Windows 有效。
- **对比报告**：报告页新增「对比报告」面板，可选**两个时段并排比较**（如「本周 vs 上周」「今天 vs 某一天」，也支持各自的自定义区间）。并排展示上屏汉字、不同汉字、活跃时长、平均速度、修改率、主力应用，并给出 A 相对 B 的升降幅度，以及两个时段各自更突出的词。
- **词库使用·词语报告**：报告页「词库使用」面板从「每个词库只显示出现最多的一个词」升级为**列出每个词库出现最多的前若干个词**（带次数、可点击跳到搜索），更像一份词语层面的报告。
- **完整报告改版·卡片叙事 + 词汇成长**：「完整报告」从一长列榜单改为更有故事感的卡片流：①**主卡**把字数换算成有体感的单位（≈ N 条微博 / N 页 A4 / N 个中篇）并标注较上期升降；②**与上期对照**卡（本周 vs 上周 / 本月 vs 上月 等，复用对比报告，自定义区间无基线时自动隐藏），并排上屏字数、不同汉字、活跃时长、平均速度的升降；③**词汇增长曲线**——这段时间累计写过多少个不同词；④**词长构成环图**——双字 / 三字 / 四字及以上词的占比，配不同汉字 / 不同词语 / 本期新词三个数字；⑤**新词时间线**——把本期第一次写下的词按日期排成一条可点的时间线（点词跳搜索）。原有的新词 / 回归词 / 高频双三字词 / 长词榜 / 词性 / 主题关键词仍在下方保留。
- **介绍页入口**：标题栏在「发布页」旁新增「介绍页」按钮，一键打开项目官网（介绍功能与下载）。
- **饼图下载图例合成 + 百分比开关**：保存任意占比饼图为图片时，会把**饼图与图例（色块 · 名称 · 次数 · 百分比）合成到同一张图**——此前词库占比饼图只导出圆环、丢了右侧文字。设置页新增「饼图下载·包含百分比」开关，可选择图例是否标出具体百分比。
- **下载完成提示**：图表 / 报告卡片保存完成后，右下角弹出一个提示气泡，显示保存路径；**点击即用系统文件管理器定位到该文件**。

### Changed 调整
- **默认静默启动**：DuckType 现在默认**安静地在托盘启动**，不再自动弹出仪表盘；想保持启动即打开，在设置页勾选「启动即打开仪表盘」即可（该项原就存在，仅默认值改为关闭）。
- **设置即改即存**：设置页的任意改动（每日目标、开关、热键、黑名单等）现在**改完即自动写入**，无需再翻到底部点「保存设置」；改每日目标后看板目标环也会立刻按新目标刷新（写入即让相关缓存失效）。底部仍保留「保存设置」按钮作为手动触发。
- **词库占比·滑条按需显示**：词库占比每张卡片的「显示词数」滑条、「显示其他」与下载按钮，默认**淡出隐藏**，只有当鼠标移到该词库卡片上（或控件获得焦点）时才显现，读饼图时不再被控件干扰。
- **随身鸭排版修复**：「本次会话」与「今日字数」本就并排在同一行、并不占额外高度，旧逻辑却按窗口高度把「今日字数」藏掉，导致缩小时只剩孤零零一格、右侧大片留白。现在**两格始终成对显示**（任何尺寸都能同时看到本次会话与今日字数）；随窗口变矮改为依次精简真正纵向堆叠的装饰行——时钟日期 → 速度小曲线 → 目标条 → 时钟 → 最后才缩小仪表盘，速度表与两格字数始终保留，下方不再单薄。
- **词库占比·可显示更多词**：词库占比饼图的「显示词数」滑条上限由 30 提升到 **100**，后端一次取回更多词，长词库也能看到更靠后的词。
- **统一空状态与加载骨架**：新增一组共享的「空状态 / 加载骨架」基元（`emptyHTML` / `skelChart` / `skelLines`），新面板（使用历史、对比报告、词库词语报告）统一使用，观感更一致。
- **后端 `stats.py` 拆分**：把「生僻字判定 + 趣味榜单」抽到独立的 `analysis/fun.py`，新的「对比报告 + 使用历史」分析放进独立的 `analysis/reporting.py`，`stats` 仍按原名重导出，调用方与测试不受影响，核心统计模块更聚焦、易维护。
- **随身鸭更名**：原「迷你计数器」正式更名为 **随身鸭**（一只随身、置顶、显示码字速度的小鸭）；全站文案、托盘小窗标题、设置项、使用历史统一改用新名。
- **最大化适配任意屏幕与多显示器**：窗口最大化改为**填满当前所在显示器的工作区**，并按该显示器的 DPI 缩放换算逻辑像素——此前固定取主显示器、又未做 DPI 换算，于是在 125%/150% 缩放下会溢出、把窗口拖到副屏后最大化也只按主屏尺寸。现在无论笔记本屏还是外接显示器、任意缩放比例都能正确铺满。页面底部仍保留一条**状态摘要条**（当前范围字数 / 不同汉字数 / 起始记录日期 + 本地存储说明），用 `margin-top:auto` 钉在内容底部填补剩余空间。
- **随身鸭·本次时长**：随身鸭最底部新增一行「本次时长」，记录的是**本次随身鸭打开后已经过去的时长**（从 0:00 起按墙钟实时累加，不足 1 小时显示 分:秒，超过显示 时:分:秒），每次重新打开随身鸭归零，而非跟随打字会话；随窗口变矮在目标条之前一并精简。
- **完整报告·主卡换算更丰富**：主卡的字数换算从固定的「微博 / A4 / 中篇」扩展为一组更有文学体感的单位（短信、朋友圈、现代诗、手写信、公众号推文、散文随笔、短篇 / 中篇 / 长篇小说、学位论文等），并在每次生成报告时**随机挑选其中几项**呈现；主卡字体排版同步细化（数字更醒目、换算项改为圆角标签）。
- **图表下载按钮修复**：各图表的「⬇ 保存为图片」在原生窗口里点击无反应（WebView2 不响应 `data:` 链接下载）。现在改为经原生桥接弹出系统保存对话框，与数据导出一致；浏览器预览仍走链接下载。
- **词库占比图例排版**：图例每行的「颜色 + 词」与右侧「次数 / 百分比」之间不再留出大片空白——词按自身宽度紧贴数字，长词省略号截断，空白收到行尾。饼图与图例进一步改为**一组居中的紧凑布局**（图例按内容宽度、不再撑满整块），饼图不再贴着最左、中间也不再空旷。
- **看板目标环·渐变多圈到 9999%**：今日目标环从单色填充改为**有方向感的渐变**——弧线顺时针由深到亮扫过，前端带一抹高亮「头部」标出当前进度的位置；超过 100% 时整圈**换一种色相重新填充**（第一圈金、第二圈绿、第三圈青……相邻圈明显不同），最高支持到 9999%。
- **使用历史·随身鸭命名对齐**：仪表盘使用历史的最近足迹标签与每日柱状图图例由「迷你」统一改为「随身鸭」，与卡片统计口径一致。
- **序列页头部重排**：输入序列页顶部从一排单调的导出按钮改为更清爽的工具栏（筛选在左、分段式「导出为 TXT / CSV / JSON」在右），段数统计移到标题旁的小标签。
- **序列分页（告别 300 条硬上限）**：序列不再一次性塞满（旧版硬截 300 条且全量渲染会拖慢滚动），改为分页渲染——先显示一页，「加载更多」按需追加，DOM 不会一次堆上千条；关键词命中处在段落里高亮。
- **活跃热力图对齐**：表格改为固定布局并给表头角格补上占位，顶部的小时刻度从此**精确落在对应列上**，整张热力图不再歪斜。
- **返回顶部按钮**：浮动「返回顶部」从紧贴右下角略微内移；当鼠标**靠近**时自动虚化、并让出点击（不再遮挡下方数据），只有真正悬停其上才恢复为可点击的实心按钮。
- **示例数据隐私核查**：核查演示模式所用示例数据，确认其语料、应用名与人名（张小满等）均为虚构、不含任何真实个人信息，无需重新生成。

## [0.2.7] - 2026-06-20

### Added 新增
- **迷你计数器（悬浮置顶小窗）**：看板左下「返回顶部」上方新增「迷你计数器」（一张较大的透明插画，在亮 / 暗两套配色下各有三张，缓慢轮换、随主题切换）。点开后主窗收起、弹出一个始终置顶的小窗，以一只**无刻度仪表盘**指针实时显示输入速度，并配一条近一分钟的**速度小曲线**、本次会话字数、今日字数与今日目标进度；关闭小窗即回到看板。速度改为**按最近输入的时间衰减加权**估算（连续、不再以 6 字/分为台阶跳变，单个孤立字符记为 0），指针跟随**平滑后**的速度移动，当前速度下方另显示该平滑值。小窗整体收小、去掉了顶部「码字鸭」标题栏（仪表盘区域即可拖动，关闭按钮浮于右上角）。
- **词库查看 / 编辑子窗口**：词库每一行新增「查看」。点开一个应用内弹窗（背景变暗、居中卡片、可按 Esc 或点遮罩关闭），能搜索、分页浏览该词库的全部词条；自建词库还能在弹窗里直接增删词并保存。
- **常用字表可查看 + 额外常用字**：新增内置「常用字」词库（约 3500 常用字 + 现代补充，默认停用、只读），可在其弹窗里查看；并提供「额外常用字」清单——把你常写、不希望算作生僻字的字（人名 / 术语用字等）加进去即可。
- **词汇丰富度趋势**：看板新增一张按日折线，展示每天「不同汉字 ÷ 总字数」，直观看出用字是更丰富还是更重复。
- **年度贡献热力图**：看板新增仿 GitHub 的近一年每日字数格子图（颜色越深写得越多），始终展示最近约 52 周、不随顶部时间范围变化，点任意一格可跳到那一天。

- **关注词**：搜索页新增「关注词」面板，可以把人名、项目代号、口头禅等想长期留意的字词加入关注。DuckType 会直接在上屏序列里逐字精确统计这些词的出现次数、最近使用、出现天数和常用程序，不依赖分词，因此 jieba 切不准的人名和缩写也能数准。每张关注卡片随顶部时间范围实时更新，点击即可查看该词的详细时段、程序与上下文分布。
- **关注词进入词频体系**：关注词会同时作为自定义词加入分词器，于是它们也能正常出现在「高频词 / 词性 / 主题」里，而不只是单独计数。改动或新增关注词后，历史数据的词频汇总会自动重建以应用新切分。
- **关注词分组与趋势**：关注词支持可选分组（如「同事」「项目」），卡片按分组归类展示；每张卡片新增一条迷你趋势曲线（按日）和与上一周期的环比箭头。
- **演示数据模式**：新增「加载演示数据」入口（看板空状态提示与设置页各一处）。会用一份示例数据预览仪表盘的完整效果，**不会改动你的真实记录**，退出即恢复，方便初次了解 DuckType 能展示什么。
- **按小时分布图样式切换**：「按小时输入分布」面板新增「柱状 / 平滑」切换，可在条形图与平滑曲线之间选择，选择会被记住。
- **分享长图**：报告页除方形卡片外，新增「保存长图」。长图包含每日趋势、行为洞察、高频词以及按应用（项目维度）的输入分布，更适合做周报长图或年度回顾。
- **数据导出 / 导入（整库迁移）**：设置页「数据管理」新增「导出全部数据 / 导入数据」。导出会把**全部数据**（数据库 + 配置）打包成一个 `.duckpack` 文件用于备份或换电脑；导入会用该文件**覆盖当前全部数据**，方便整体迁移。

  连续天数、高峰时段、距下一个成就、"上次打出某个词是多久以前、之间又码了多少字"等）+ 你自己的句子；
  右侧 🔀 可随时换一句，并会偶尔自动刷新。趣味 / 文学 / 冷知识句子放在可自行编辑的
  `%APPDATA%\DuckType\phrases.txt`（首次运行自动生成示例，每行一句、`#` 开头为注释）。

### Changed 调整
- **词性分类更易懂、更准确**：① 词性分布图从 jieba 的约 40 个细分标签（名语素、副形词、时态助词(着)…）
  归并为 9 个**大类**（名词 / 动词 / 形容词·状态 / 副词 / 代词 / 数量词 / 时间·方位 / 虚词 / 其他），
  点进某大类再看其下的具体词语；② 修正 jieba 自带词典对部分常用词的明显误标（如「位置 / 东西」被标成
  动词 / 地名、「可以」被标成连词），用一张小型高置信度更正表纠正，更改后历史数据会自动重新分词生效
  （`seg_version`）。词性体系沿用 jieba 的 ICTCLAS（北大）标注集，本身为机器自动标注、并非 100% 准确。
- **可点击元素的悬停反馈**：标签页、按钮、主题词、可点击的条形 / 扇区等在悬停时会有边框高亮、轻微上浮
  或手型光标，"可以点" 的地方一眼就能看出来。
- **放宽「生僻字」判定**：旧版只把 CJK 扩展区 / 兼容区的字算作生僻字，几乎永远为空。现改为以
  《现代汉语常用字表》3500 常用字为基准——凡不在该字表内的汉字即视为生僻字（按标准字表判定，
  不依赖你输入了多少次），既更容易上榜，又不会把日常常用字算进去。

### Changed / Fixed 优化与修复
- **浅色主题可读性**：修复浅色模式下横幅（提示/趣味）文字沿用深色配色导致几乎看不清的问题（加粗文本
  曾是白色、警告文字接近纯白）；热力图色调、字体颜色/层级随主题正确切换，并加入字体抗锯齿，整体在
  白天模式下更清晰。
- **修复 QQ 等程序把占位文字误记为输入**：群聊被全员禁言时输入框直接显示的「全员禁言中」等占位/提示
  文字（并非用户键入）会被混进序列。根因是这类文字由控件自行写入、并未经过输入法组合（composition）。
  现在通过 TSF 组合生命周期回调（`ITfContextOwnerCompositionSink` 的 OnStart/OnEndComposition）判断
  该文档是否真的发生过输入法组合，只有发生过才采信其上屏文本；该回调在微信 / 资源管理器 /
  Electron（如 Claude）等**不暴露 composing 区段**的程序里同样会触发，因此既挡掉占位文字、又不会漏记
  这些程序的真实输入；若某程序无法挂上该回调，则自动放行（fail-open），绝不静默丢字。
- **修复更新后重复计数无需重启电脑**：旧机制每次更新都要靠改协议版本号让上一版被 pin 住的钩子失效，
  而同版本的多份钩子仍会叠加计数（开发期重复构建即出现 2×/3×）。现在每个宿主进程内的多份钩子会按
  「宿主代次」选出**唯一上报者**（宿主窗口每次启动都会换，作为代次令牌，用进程内命名共享段协调），
  宿主重启后自动重新推选——更新 / 重复注入都不再重复计数，也不必重启电脑。（同时把协议版本升到 V6
  以淘汰早于该机制的历史钩子。）
- **修复输入速度被算成天文数字**：输入法常把整词的多个字几乎同一瞬间上屏，旧算法用「相邻字时间差之和」
  作为有效时长，遇到这种成簇上屏时分母趋近 0，平均/峰值速度会暴涨到上万甚至十几万 cpm。现改为按
  「会话时间跨度」计速，并对每个字设下极小的最短耗时下限（成簇上屏时把表观速度封顶在约 500 cpm 的
  人类可达上限），正常节奏的输入完全不受影响。
- **饼图 / 条形图交互反馈**：词性饼图悬停时对应扇区会弹出并描上白色高亮环、鼠标变为手型；可点击的
  高频字 / 高频词条形悬停时整条以品牌色高亮，选择 / 点击目标更醒目。

## [0.1.3]

### Fixed 修复
- **修复拼音长词分段选择导致序列重复（`ABABCD`）**：根因是把「候选栏/预览里尚未上屏的组合文本」
  也计入了统计。整句/联想输入时先选定 `AB` 这个词，`AB` 其实仍停留在候选区、并未落到文档，
  IME 把不断增长的组合（`AB`→`ABCD`）作为编辑回报，于是被算成 `AB`+`ABCD`。现在 TSF 捕获会
  借助 `GUID_PROP_COMPOSING` 跳过仍处于组合状态的文本，**只在最终上屏（组合结束、不再 composing）
  时计入一次**，从根上消除重复，也不再需要任何「事后去重」的启发式。捕获时只枚举 composing 属性
  变化（不带 `TF_GTP_INCL_TEXT`），避免「上屏时重插文本」的输入法把同一个词记两遍。
- **修复平均/峰值速度统计**：输入法整词上屏会让一个词的每个字共用同一时间戳（我们只拿得到「上屏
  时刻」，没有逐字时刻）。旧版按字间隔算瞬时速率会被 0 间隔除爆，峰值一度高达数百万字/分。现在
  ① 峰值速度＝「任意 60 秒窗口内的最高字数」（你最高产的一分钟），有界且真实；② 平均速度＝总字数
  ÷活跃分钟，活跃分钟为会话内相邻上屏间隔之和，去掉了旧版「每个会话至少按 60 秒计」导致平均值
  系统性偏低的问题。
- **修复退格/删除率**：修改率只统计「删掉刚敲下不久的汉字」——已经停留一段时间的汉字视为“已落定”，
  之后长按退格删除其它内容（如未被捕获的英文）不再被计入删除率。
- **修复更新后重复计数（2 倍/3 倍）**：旧版钩子 DLL 会被 pin 在每个宿主进程里直到该进程退出，
  升级后若窗口/消息协议名不变，这些残留 DLL 会继续把上屏文本回传给新实例，导致同一句话被记 2~3 次。
  现在把宿主窗口类与消息名升级到 `V5`，旧 DLL 找不到新窗口即自动失效（今后改动捕获逻辑都应同步 bump）。
- **修复一键更新**：① 更新脚本原来用 `DETACHED_PROCESS` 启动、丢了控制台，`start` 拉不起新版本——
  改成隐藏控制台（`CREATE_NO_WINDOW`），用 `ping` 代替 `timeout`，并对替换动作重试（等 PyInstaller
  引导进程释放文件锁），新版本可靠重启；② 下载后校验大小（<1MB 视为重定向/错误页，直接取消，
  不会用坏文件覆盖）；③ 界面会显示新版本的**下载路径、大小与目标路径**，退出延时放宽到 3.5 秒。
